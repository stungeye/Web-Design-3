Brochure Website Planning

Overview

In Assignment 2, you will build a desktop-focused brochure-style website using semantic HTML, accessible markup, and a beautiful grid layout.

Before you dive into the markup / css you need a clear plan for the site's content, structure, accessibility, and visual direction.

For this assignment, you will plan a two-page brochure website for a real place on Earth. Your chosen place could be a city, park, attraction, historic site, travel destination, or some other exciting location.

This is an exercise in "becoming one with the grid."

Check out the visuals at the bottom of this document for some gridspiration.

Learning Outcomes

You will be assessed on your ability to:

  • Plan a small website with a clear purpose, audience, and content structure.
  • Organize two connected pages with clear headings, sections, and navigation.
  • Apply semantic HTML thinking before beginning final coding and styling.
  • Identify basic accessibility considerations related to headings, links, images, colour, and keyboard use.
  • Use Figma to communicate a realistic desktop visual direction for a website that will later be built in HTML and CSS.
  • Plan desktop layout sections using basic CSS Grid thinking, including containers, columns, spacing, and major content groupings.
  • Credit image and text sources clearly.

Assignment Deliverables

For this assignment, you will submit two main items:

  1. A Website Planning Document
  2. Figma Mockups

Your planning document explains the content, structure, accessibility, desktop layout and grid direction, and source credits for your website. Your Figma mockups show the visual direction for the two pages you plan on building in Assignment 2.

Location Selection

Choose a real place on Earth for your brochure website.

Your location could be something like:

  • A city, town, or region
  • A national/provincial park or natural landmark
  • A museum, gallery, or historic site
  • A travel destination or attraction

Your chosen location should give you enough visual and written content to create two connected brochure-style pages.

Important Scope Note

Assignment 1 and Assignment 2 are focused on planning and building a strong fixed-size grid-based website. 

You are not required to create a mobile mockup or a responsive design strategy for this assignment. Responsive design will become a major focus later in the course.

Website Planning Document

Create a professionally written planning document with the following sections:

1. Location and Website Goal

This introductory section should include:

  • The name of the location
  • Where it is located
  • Why you chose it
  • The intended audience for the website
  • The main mood, theme, or story you want the website to communicate
  • The main goal of the website

2. Website Content Plan

Your planned website must include two pages.

Website Page 1: Main Brochure Page

This page should introduce the location and establish the main story, mood, or purpose of the site.

For example, this page might answer:

  • Why should someone care about this place?
  • What kind of experience does it offer?
  • What makes it visually, culturally, historically, or geographically interesting?

Website Page 2: Supporting Detail Page

This page should continue from the call to action on Page 1.

For example, this page could focus on:

  • A visitor guide
  • A featured landmark
  • Conservation or cultural information
  • History, geography, events, activities, or practical visitor information

In this section of your planning document you should include the following for each of these two pages:

  • Page title
  • Page purpose
  • Main heading
  • Planned sections
  • Planned subheadings
  • Planned paragraph topics with draft content
  • Planned images or illustrations
  • Call-to-action text on Page 1 that leads logically to Page 2

Your planned website should be large enough to support a meaningful Assignment 2 build. As a general target, plan at least four meaningful content sections for Page 1 and at least three meaningful content sections for Page 2. Across the full website, plan at least five meaningful images or illustrations, with at least one visual on each page.

3. Semantic HTML Structure Plan

In this section of the planning document, create a simple semantic structure plan for each page. You are not expected to write full code yet. The goal is to show that you are thinking about page structure before visual styling.

Your semantic plan may be written as a nested list or simplified HTML skeleton.

Your plan should identify where you expect to use elements such as:

  • header
  • nav
  • main
  • section
  • aside
  • article
  • figure + figcaption
  • blockquote + cite
  • details + summary
  • footer
  • h1, h2, h3, and other headings as needed
  • Links or buttons

You are not expected to use every element in the list, and you can use elements outside those listed. Choose elements because they fit the meaning and structure of your content, not because you want to include more tags.

Assignment 2 will require a shared header and navigation that allows users to move between both pages. Plan accordingly.

4. Accessibility Notes

Your planning document should include accessibility notes for your planned website, addressing:

  • Heading hierarchy
  • Link and button text
  • Draft alt text for meaningful images
  • Images that may be decorative
  • Colour contrast concerns (Research colour contrast accessibility guidelines.)
  • Any content that might need extra care to be understandable

You do not need to complete a full accessibility test for Assignment 1. The goal is to identify basic accessibility decisions early so they can guide your Assignment 2 build.

5. Layout and Grid Notes

Your planning document should explain the layout direction for each page. You are not expected to write final CSS yet, but you should show that you are thinking about how your content could be arranged using CSS Grid, Flexbox, and simple block layout.

For each major page section, identify:

  • Whether the section will likely use CSS Grid, Flexbox, or simple block layout
  • The main content groups inside the section
  • The approximate desktop column structure, such as one column, two columns, three columns, or a card grid
  • How the header and navigation will be arranged
  • How the hero or introduction area will be arranged, if applicable
  • How images, text, navigation, cards, or calls to action should align
  • Where consistent spacing or gaps will be important
  • Any layout choices that should preserve a logical reading order in the HTML

For example, you might note that a hero section will use a two-column grid with text on one side and an image on the other, or that a featured-items section will use a three-column card grid.

Be sure to pick a target fixed width that will support your grid design. The recommended range is 1440px to 1920px.

You do not need to solve every CSS detail yet. The goal is to show that your desktop layout has been considered before you begin building it.

6. Image and Text Source Credits

Include credits and URLs for any images, text sources, or references used in your mockups or planning.

You may use images from sources that clearly allow reuse for student, educational, or portfolio-style work. You must keep track of where your images came from.

Acceptable image sources include:

  • Your own photos
  • Public domain images
  • Creative Commons images, as long as you follow the license requirements
  • Free stock image sites such as Unsplash, Pexels, or Pixabay
  • Official tourism, museum, city, park, or landmark websites only when the image usage rights are clearly stated and allow reuse

Do not use images copied from Google Images, social media, travel blogs, news articles, or random websites unless you can verify that the image license allows reuse.

For each planned image, include:

  • The image source
  • The image URL
  • The creator or organization, if listed
  • The license or usage permission, if listed
  • Draft alt text, or a note that the image is decorative

Figma Mockups

Create Figma mockups that show the visual direction for your two-page brochure website.

Your Figma work must include:

  • One fixed-width desktop mockup for Page 1
  • One fixed-width desktop mockup for Page 2

Your mockups or wireframes should show:

  • Desktop layout
  • Typography direction
  • Colour direction
  • Image usage
  • Navigation
  • Call to action from Page 1 to Page 2
  • Visual hierarchy
  • Approximate spacing and alignment
  • Brief layout annotations, such as "two-column hero grid," "three-card grid," "content container," "full-width section," or "flex navigation"

You are not required to create a mobile mockup for this assignment.

Assessment Guidelines

This assignment is about creating a clear, useful plan for the two-page brochure website you will build in Assignment 2. A strong submission should do more than include the required pieces. It should make specific decisions about content, structure, accessibility, desktop layout/grid direction, and visual direction. It each is marked out of 4.

A category mark of 3 represents complete, appropriate work that is ready to move forward, but may still be somewhat basic/generic or leave some design/build decisions unresolved.

A category mark of 4 represents planning that goes beyond completion. It should make clear, specific, connected decisions that reduce uncertainty and would make the Assignment 2 build noticeably stronger.

1. Website Concept and Content Plan

This category evaluates how clearly you have planned the purpose, content, and two-page structure of your brochure website.

Evidence of Success

  • The location is real and suitable for a brochure website
  • The website has a clear purpose and intended audience
  • Page 1 and Page 2 have distinct but connected roles
  • Headings, sections, and paragraph topics are specific enough to build from
  • Planned images support the content
  • The Page 1 call to action leads naturally to Page 2
  • The plan reduces major content decisions later in Assignment 2

2. Semantic HTML and Implementation Planning

This category evaluates how well you have planned the structure of your pages before writing final code.

Evidence of Success

  • Each page has a planned structure
  • header, nav, main, and footer are used appropriately
  • Sections are based on meaningful content groups
  • Heading levels follow a logical hierarchy
  • Links and calls to action have a clear purpose
  • Images are placed in meaningful context
  • Elements such as figure, figcaption, aside, article, blockquote, cite, details, or summary are used only where appropriate
  • The plan describes content structure, not just visual layout

3. Accessibility and Desktop Layout/Grid Planning

This category evaluates how well you have planned for basic accessibility and desktop layout before building the site, including whether you are beginning to think in terms of Grid-based layout systems.

Evidence of Success

  • Heading hierarchy is considered
  • Link and button text will be clear
  • Meaningful images have draft alt text
  • Decorative images are identified
  • Colour contrast is considered
  • Major desktop layout sections are identified
  • CSS Grid is considered for major layout areas where appropriate
  • Flexbox is considered for navigation, rows, shelves, cards, or smaller alignment tasks where appropriate
  • Approximate desktop column structures are described
  • Navigation placement is clear
  • Image placement supports the content
  • Spacing and gaps are considered as part of the layout system
  • The layout plan is realistic to build in Assignment 2

4. Figma Mockups as a Build Plan

This category evaluates whether your Figma work clearly communicates the desktop visual direction for the website you will build.

Evidence of Success

  • Desktop mockup for Page 1 is included
  • Desktop mockup for Page 2 is included
  • The two pages feel like the same website
  • Layout supports the planned content
  • Navigation and call to action are clear
  • Typography supports readability
  • Colour and image choices support the intended mood
  • Spacing and alignment are intentional
  • Major layout areas are labelled or easy to identify
  • Grid, Flexbox, or block layout choices are annotated where helpful
  • The mockups or wireframes match the written layout notes
  • The design appears realistic to build with HTML and CSS

5. Sources, Credits, Professionalism, and Submission Readiness

This category evaluates the overall clarity, organization, and completeness of your submitted planning package.

Evidence of Success

  • Planning document is submitted as PDF or DOCX
  • Figma mockups or wireframes are exported as PDF
  • Figma share link is included and accessible
  • Files are organized clearly in the ZIP
  • Written work uses headings and readable formatting
  • Image sources and URLs are included
  • Image creator and license are provided, if available
  • Draft alt text or decorative-image notes are included
  • Planning document and mockups match each other
  • The full package is ready to support Assignment 2
