# Readable Defaults

Semantic HTML gives a page meaning.

Your "Base" CSS makes a page readable, usable, and ready for layout.

This week is about the layer of styling that should happen before grids, cards, hero sections, or polished visual systems.

<Note>
  Styling starts with readable defaults. If the base page is hard to read, the
  layout will only make the problem more decorative.
</Note>

## From Skeleton To Reading Experience

Last week we built the page skeleton:

```text
body
  header
  main
    h1
    sections
  footer
```

Before we move to layout, we build the base reading experience.

<Question>
  What base styles would make for a comfortable reading experience?
</Question>

## The Default Browser Page Is A Starting Point

Browsers already give HTML some readable behaviour:

- headings are larger than paragraphs and hierarchically sized
- lists have indentation
- links are visually distinct
- buttons can be focused and activated
- images keep their natural dimensions

Those defaults are useful, but they are not a finished design.

## A Base Stylesheet Has A Job

A base stylesheet sets the ordinary rules that should be true across the site.

It usually handles:

- box sizing
- body margin and text rendering
- default font family, line height, and text colour
- heading rhythm
- paragraph and list spacing
- link and focus styles
- image and media behaviour
- simple form and button readability

This layer is not where we solve the full page layout.

<Warning>
  A reset or base style should not erase useful behaviour unless you
  intentionally replace it.
</Warning>
## Reset Less Than You Replace

Modern CSS resets are useful because they smooth out awkward defaults, not because they remove every browser style.

A good base stylesheet should:

- keep useful semantic behaviour
- remove defaults that make layout awkward or unpredictable
- improve readability
- give media and form controls nicer defaults
- leave room for the actual design layer

<Practice>
  Before removing a browser default, decide what better default will replace it.
</Practice>

## Start With Box Sizing

The first common base rule changes how width and height are calculated.

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

With `border-box`, an element's declared width includes its padding and border. This makes layout math easier once Grid and Flexbox enter the project.

## Body Defaults And Readable Type

The `body` rule controls the first impression of the whole reading experience.

```css
body {
  margin: 0;
  font-family: system-ui, sans-serif;
  color: #1f2933;
  background-color: #fffdf8;
}
```

Readable type is not only about choosing a nice font.

These body choices affect every paragraph, list item, heading, and link on the page:

- `font-family` chooses the voice of the page
- `color` and `background-color` decide whether the text is easy to see
- the base font size gives `rem` units their starting point

<Practice>
  Choose body defaults that make content pleasant to read before adding
  section-specific styling.
</Practice>

## The Default Font Size

If you do not set a font size, most browsers use a default of `16px` for ordinary body text.

```css
body {
  font-family: system-ui, sans-serif;
  /* font-size defaults to 16px */
}
```

Leaving the base size alone is often a good starting point. Users can change their browser defaults, and `rem` units will respect that choice.

## Units For Readable Defaults

CSS units describe relationships. Learn what each unit measures before choosing where to use it.

- `px` is a fixed CSS pixel
- `rem` is relative to the root font size
- `em` is relative to the current element's font size
- `ch` is based on the width of the `0` character in the current font

Those relationships make each unit useful for different base-style decisions:

- use `px` for small fixed details like borders and focus outlines
- use `rem` for spacing and type sizes that should relate to the page's base font size
- use `em` when a value should scale with the element it belongs to
- use `ch` for readable line length

```css
:focus-visible {
  outline: 3px solid #f59e0b;
  outline-offset: 0.2rem;
}

main {
  max-width: 70ch;
  padding: 2rem;
}
```

<Practice>Pick units based on what the value should respond to.</Practice>

## Form Controls Should Match The Page

Inputs, buttons, textareas, and selects do not always inherit the page's font styles by default.

```css
input,
button,
textarea,
select {
  font: inherit;
}
```

This small rule helps form controls and buttons feel like part of the same site instead of a separate browser widget.

<Note>
  Inheriting the font does not remove the need to further style buttons and
  forms. It only gives them a better starting point.
</Note>

## Limit The Reading Measure

The `main` element is a good place to control the readable measure of a simple notes-style page.

```css
main {
  max-width: 70ch;
  margin-inline: auto;
  padding: 2rem;
}
```

These rules do three different jobs:

- `max-width: 70ch` keeps text lines from becoming too long
- `margin-inline: auto` centers the content block when there is extra space
- `padding: 2rem` keeps text away from the viewport edge

The `ch` unit can help limit text-heavy content because it is based on the width of the `0` character in the current font.

<Warning>
  Do not use one `main` rule as the final layout for every site. It is a
  readable starting point for simple pages, not a full layout system.
</Warning>

## Line Height Choices

Different text sizes need different amounts of line height.

```css
* {
  line-height: calc(1em + 0.5rem);
}
```

This formula combines two useful relationships:

- `1em` follows the current element's font size
- `0.5rem` adds breathing room based on the root font size

The result is flexible. Smaller text gets enough extra space to read comfortably. Larger headings get less extra space relative to their size, so they stay visually grouped.

If the base font size is `16px`, a `16px` paragraph gets a line height of about `24px`.

## Scale The Heading Hierarchy

Browser heading sizes are useful for raw HTML, but a finished site should usually choose its own heading scale.

Most browsers start with heading defaults similar to this:

{/* prettier-ignore */}
```css
h1 { font-size: 2em;    }
h2 { font-size: 1.5em;  }
h3 { font-size: 1.17em; }
h4 { font-size: 1em;    }
h5 { font-size: 0.83em; }
h6 { font-size: 0.67em; }
```

These defaults create visible hierarchy, but the lower levels can become smaller than body text. That is not always the most readable choice for a designed page.

You can hand-pick heading sizes:

{/* prettier-ignore */}
```css
h1 { font-size: 2.5rem; }
h2 { font-size: 1.75rem; }
h3 { font-size: 1.25rem; }
```

Hand-picked sizes are fine when you have a clear visual reason. Another approach is to choose a type scale, where each heading step grows by the same ratio.

Common scales include:

- Minor Third, `1.200`, subtle
- Major Third, `1.250`, balanced
- Perfect Fourth, `1.333`, slightly more dramatic
- Perfect Fifth, `1.500`, bold
- Golden Ratio, `1.618`, bolder

## A Balanced Heading Scale

A Major Third scale multiplies each step by `1.25`.

```css
h1 {
  font-size: 1.953rem;
}

h2 {
  font-size: 1.563rem;
}

h3 {
  font-size: 1.25rem;
}
```

With a `16px` root font size, this gives approximately:

- `h1`: `31px`
- `h2`: `25px`
- `h3`: `20px`
- body text: `16px`

The exact numbers can change from project to project. The important part is that the scale supports the content outline.

<Practice>
  Style headings to show hierarchy. Do not choose heading levels because you
  want a certain visual size.
</Practice>

## Spacing Creates Rhythm

Base spacing should help the reader understand what belongs together.

```css
section > * + * {
  margin-block-start: 1rem;
}
```

This is often called the owl selector because `* + *` looks a bit like two eyes.

It means:

- find every direct child inside `<main>`
- skip the first child
- add space before each following sibling

```html
<section class="flow">
  <h2>Hours And Admission</h2>
  <p>The park is open daily.</p>
  <p>Special events may require tickets.</p>
</section>
```

This lets the parent control the spacing rhythm of its children.

## Spacing Variants

Different content groups can have different rhythms.

```css
article > * + * {
  margin-block-start: 1rem;
}

section > * + * {
  margin-block-start: 2rem;
}

main > * + * {
  margin-block-start: 4rem;
}
```

This is usually cleaner than giving every paragraph, list, heading, and section its own bottom margin.

<Warning>
  Margins on individual elements can still be useful. The key is to avoid random
  one-off spacing that fights the page rhythm.
</Warning>

## Links Need Clear Text And Clear Styling

Accessible links need both meaningful words and intentional visible affordance.

```html
<a href="hours.html">View visitor hours and admission details</a>
```

```css
a {
  color: #075985;
  text-decoration-thickness: 0.08em;
  text-underline-offset: 0.16em;
}

a:hover {
  color: #0f766e;
}
```

<Practice>
  A link should make sense when read away from the text around it.
</Practice>

Avoid vague link text such as:

- click here
- read more
- learn more

## Focus Is Part Of The Design

Keyboard users move through interactive elements using focus.

```css
:focus-visible {
  outline: 3px solid #f59e0b;
  outline-offset: 0.2rem;
}
```

Do not remove outlines just because they look unfamiliar.

```css
/* Avoid this */
*:focus {
  outline: none;
}
```

<Warning>
  Removing focus styles can make a page impossible to use without a mouse.
</Warning>

## Images Need Defaults Too

Images can easily overflow their containers if the base CSS does not constrain them.

```css
img,
picture,
canvas,
svg,
video {
  display: block;
  max-width: 100%;
  height: auto;
}
```

This rule does not decide the final image layout. It makes media safer by default.

<Aside>
  A future interactive demo could let students toggle image defaults on and off
  to see how an oversized image can break an otherwise sensible page.
</Aside>

## Alt Text Is A Content Decision

Alt text explains the purpose of an image for people who cannot see it or when the image does not load.

Good alt text depends on context.

```html
<img
  src="the-leaf.jpg"
  alt="The Leaf's glass roof rising above the surrounding gardens"
/>
```

If an image is decorative and adds no useful information, it can use an empty alt attribute.

```html
<img src="divider-pattern.svg" alt="" />
```

<Practice>
  Alt text makes images accessible to folks using screen-readers.
</Practice>

## Buttons And Links Are Not The Same Job

Use a link when the user goes somewhere.

```html
<a href="visit.html">Plan your visit</a>
```

Use a button when the user triggers an action on the current page.

```html
<button type="button">Show ticket options</button>
```

They can look visually related, but they should not be swapped just for styling convenience.

## A Small Base Stylesheet

This is a reasonable beginning for a small brochure site:

{/* prettier-ignore */}
```css
*, *::before, *::after {
  box-sizing: border-box;
}

* {
  line-height: calc(1em + 0.5rem);
}

body {
  margin: 0;
  font-family: system-ui, sans-serif;
  color: #1f2933;
  background-color: #fffdf8;
}

input, button, textarea, select {
  font: inherit;
}

/* Minor Third Scale 1.25x */
h1 { font-size: 1.953rem; }
h2 { font-size: 1.563rem; }
h3 { font-size: 1.25rem;  }
h4 { font-size: 1rem;     }

img, picture, canvas, svg, video {
  display: block;
  max-width: 100%;
  height: auto;
}

a {
  color: #075985;
  text-underline-offset: 0.16em;
}

:focus-visible {
  outline: 3px solid #f59e0b;
  outline-offset: 0.2rem;
}

/* And if we want a very basic layout with vertical rhythm: */

main {
  max-width: 70ch;
  margin-inline: auto;
  padding: 2rem;
}

main > * + * {
  margin-block-start: 4rem;
}
```

Start with the semantic HTML from last week. Do not change the structure unless you find a real accessibility or content problem.

Apply a small base stylesheet that handles:

- `box-sizing`
- `body` typography and colour
- readable units for type, spacing, and measure
- readable heading and paragraph line height
- a clear heading scale based on the base font size
- parent-controlled spacing rhythm
- clear link styling
- visible keyboard focus
- safe image defaults
